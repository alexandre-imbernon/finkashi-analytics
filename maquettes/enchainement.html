Enchainement des ecrans

Modelisation des transitions entre les ecrans du plugin Finkashi Analytics dans l'admin WordPress.

Navigation directe (menu, lien) Transition via action (bouton, validation)

Description des transitions

  1. L'administrateur clique sur l'entree Finkashi dans le menu lateral de WordPress. Le Dashboard est l'ecran par defaut.
  2. Depuis n'importe quel ecran du plugin, le sous-menu permet de naviguer vers Dashboard, Alertes ou Reglages.
  3. Sur l'ecran Alertes, le bouton "Nouvelle regle" (ou "Modifier" sur une ligne existante) ouvre une modale de saisie. La validation ou l'annulation referme la modale et rafraichit le tableau des regles, sans changer d'URL.
  4. Apres modification dans les Reglages, le bouton "Enregistrer" sauvegarde les changements et affiche une notification de succes au-dessus du formulaire.
  5. Le bouton "Tester la connexion" dans les Reglages envoie un appel a l'API et affiche le resultat sans quitter l'ecran.
